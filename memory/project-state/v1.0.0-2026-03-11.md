### Śhāstrārtha Notes (OB-28 Human Gate)

The following findings were raised during Śhāstrārtha review and deferred to post-v1.0:

1. **DNS rebinding on webhook delivery** — the SSRF blocklist checks IP at rule creation time, not at dispatch time. A DNS rebind between these can bypass the check. Mitigation: re-resolve and validate at dispatch time (P2 — post v1.0).
2. **alert_fired table has no TTL** — alert history grows unbounded. Mitigation: add a 90-day delete job or PostgreSQL partition (P2 — post v1.0).
3. **Redis rate limiter** — currently there is no per-org rate limit on `/v1/ingest`. Mitigation: add Redis token-bucket rate limiter (P1 — Sprint 04 candidate).
